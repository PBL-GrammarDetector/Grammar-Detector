# TFLS 语法纠正程序

一个基于双引擎（LanguageTool + AI）的英语语法检测与学习工具，支持文本输入/文件上传，自动识别语法错误并提供知识卡片和专项练习。

## 功能特性

- **双引擎检测**：LanguageTool（基础语法与拼写）+ DMX AI（进阶分析）
- **高亮标注**：输入框中自动标记问题文本，区分LanguageTool和AI提示
- **写作统计**：自动统计总词数、句子数、平均句长、词汇多样性等指标
- **知识卡片**：根据检测到的错误类型，动态生成对应的语法知识卡片，包含正确和错误的对比示例
- **专项练习**：每种错误类型配套 5 道选择题，点击核对按钮即时批改并显示解析
- **文件上传**：支持上传 .txt 文件进行检测
- **响应式设计**：适配桌面端和移动端，不同屏幕尺寸自动调整布局
- **快捷操作**：支持 Ctrl+Enter 快捷键触发语法检查

## 快速开始

### 直接使用

1. 在浏览器中打开 `index.html`
2. 在左侧输入框中粘贴英文文本，或点击「上传附件」按钮选择 .txt 文件
3. 点击「语法检查」按钮，或按 `Ctrl+Enter` 快捷键
4. 在右侧面板查看检测结果，点击问题类型链接可跳转到对应的知识卡片
5. 在知识卡片中查看语法讲解和示例，完成专项练习并核对答案

### 部署到 GitHub Pages

1. Fork 本仓库到你的 GitHub 账号
2. 进入仓库的 Settings → Pages 设置
3. 在 Source 中选择部署分支（通常为 `main`），保存
4. 等待几分钟后，通过 `https://你的用户名.github.io/仓库名` 访问

## 技术架构

本项目是一个纯前端单文件应用，所有代码集中在 `index.html` 中。核心技术栈为 HTML + CSS + JavaScript，无需任何构建工具或框架。

检测流程上，用户输入文本后，系统同时向 LanguageTool API 和 DMX AI API 发起请求。LanguageTool 负责基础语法和拼写检测，DMX AI 进行更深层的语法分析。两个请求并发执行，谁先返回就先展示谁的结果，另一个结果以追加形式展示。

错误分类方面，LanguageTool 的检测结果通过关键词匹配和正则表达式来识别错误类型；DMX AI 的检测结果则直接根据 AI 返回的 error_type 字段映射到对应知识卡片。两条路线互不干扰，避免了 AI 返回类型与关键词匹配规则不一致导致的误判。

知识卡片系统为每种错误类型提供语法讲解、正确与错误示例、以及 5 道专项练习题。练习题支持单选、即时批改和解析反馈。

## 支持的语法错误类型

目前支持 8 种语法错误类型的检测，每种类型都有对应的知识卡片和练习题：

- **主谓一致问题**：主语和谓语在单复数上不匹配，如第三人称单数缺少 -s
- **冠词使用错误**：a/an/the 的使用不当，包括定冠词与不定冠词的混淆
- **时态问题**：动词时态与上下文或时间状语不一致
- **拼写问题**：单词拼写错误或常见拼写混淆
- **标点符号**：标点使用错误、缺失或多余
- **情态动词/助动词使用错误**：情态动词或助动词后未接动词原形
- **代词使用错误**：代词在格（主格/宾格）、数（单数/复数）、性上不一致
- **句子语序错误**：句子结构不符合英语语序规范，或疑问句缺少倒装
- **名词使用错误**：针对未能区分可数与不可数名词的错误

如果 AI 检测到的错误类型不在以上这几种之内，会被标记为 other，显示默认的知识卡片（通用语法知识），待后续扩充。

语法错误的匹配可能不准确，我们会持续优化

## 配置说明

### LanguageTool API

使用 LanguageTool 提供的公开 API，无需额外配置即可使用。单次检测文本限制为 20KB。但想要更迅速的检测，尤其是对于AI检测，建议单次检测控制在200词以内

### DMX AI API

使用DMX API作为AI检测，目前使用qwen-flash-2025-07-28

### 自定义知识卡片

知识卡片内容存储在 `knowledgeBase` 对象中。每张卡片包含标题、语法讲解文字、正误示例数组、以及 5 道练习题（每题含题目、选项、正确答案索引和解析文字）。参照现有卡片的格式即可添加新的语法类型。

## 使用场景

- 英语学习者自查写作中的语法错误，通过知识卡片巩固语法知识
